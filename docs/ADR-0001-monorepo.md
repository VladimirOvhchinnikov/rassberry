# ADR-0001: Монорепозиторий и Go Workspaces

**Решение:** использовать монорепозиторий с несколькими модулями Go и единым `go.work`.

**Причины:**
- Быстрый рефакторинг общих контрактов и портов.
- Единые стандарты форматирования, сборки и тестов.
- Упрощённая локальная разработка без `replace` за счёт `go.work`.

**Альтернативы:** полирепозитории (отдельный репозиторий на каждое ядро/модуль).
Подойдут позже, когда интерфейсы стабилизируются и нужен независимый релизный цикл.

**Последствия:**
- Простая навигация и сборка.
- При росте числа модулей нужно будет следить за зависимостями и версионированием.
